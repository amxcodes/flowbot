# Variables Made Simple

This page is for non-technical teammates.

If you only want "what do I fill", use this file.

## Minimum setup (just to run)

You only need these two things:

1. In `config.toml`: one working AI provider
2. In `config.toml` or `.env`: your Telegram bot token (if using Telegram)

## Fast decision guide

- Want easiest start? Use Antigravity provider.
- Want Telegram bot? Fill Telegram token and your user ID allowlist.
- Want 24x7? Install service after setup.

## Copy-ready examples

### A) Smallest `config.toml` for local chat only

```toml
default_provider = "openai"

[providers.openai]
api_key = "sk-your-real-key"
model = "gpt-5-2"
```

### B) Smallest `config.toml` for Telegram + Antigravity

```toml
default_provider = "antigravity"

[providers.antigravity]
api_key = "your-antigravity-key"

[providers.telegram]
bot_token = "123456789:AA..."
allowed_users = ["123456789"]
```

## Variable cheat sheet

- `default_provider`
  - What it means: which AI backend is used by default
  - Typical value: `"antigravity"` or `"openai"`
- `providers.<name>.api_key`
  - What it means: secret key for that provider
  - Typical value: key from provider dashboard
- `providers.telegram.bot_token`
  - What it means: Telegram bot auth token
  - Where to get: `@BotFather`
- `providers.telegram.allowed_users`
  - What it means: who can talk to your bot
  - Typical value: your Telegram numeric user ID(s)
- `RUST_LOG` (optional env var)
  - What it means: logging detail level
  - Typical value: `info`

## Values people often confuse

- API key vs OAuth token:
  - API key is pasted manually.
  - OAuth token is generated by `nanobot login <provider>`.
- Telegram username vs user ID:
  - allowlist needs numeric user ID, not `@username`.
- `config.toml` vs `.env`:
  - runtime primarily reads `config.toml` for providers.
  - `.env` may be used by service/runtime helpers depending on deployment.

## Safety rules

- Never commit real keys/tokens.
- Keep `InteractionPolicy` interactive unless you intentionally want autonomous execution.
- If service runs but bot does nothing, check provider key first.

## Verify after filling values

```bash
nanobot doctor
nanobot gateway --channel telegram
nanobot service status
```
