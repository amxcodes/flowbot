## What this handbook is trying to do

- Explain the system in plain language.
- Show where code lives so contributors do not get lost.
- Document the moving parts (agent, gateways, tools, memory, cron, skills, admin API).
- Make day-1 setup and day-2 operations repeatable.

## What this handbook is not

- Not a full API reference for every function.
- Not a replacement for `docs/guides/*` deep dives.
- Not marketing copy.

When in doubt, this handbook should favor clarity over cleverness.
